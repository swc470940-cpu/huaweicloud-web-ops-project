# 运维手册

## 巡检项

- 检查 Nginx 是否运行
- 检查 WordPress 首页是否可访问
- 检查数据库连接是否正常
- 检查磁盘使用率是否正常
- 检查备份文件是否存在

## 常见问题

### 502 Bad Gateway
- 先查 Nginx 是否运行
- 再查 PHP-FPM 是否运行
- 查看 `/var/log/nginx/error.log`

### 数据库连接失败
- 先查 RDS 状态与内网地址
- 再查安全组是否允许 ECS 访问数据库端口
- 最后查 WordPress 配置文件中的数据库地址、账号、密码

### 网站访问慢
- 先看 ELB 后端服务器组健康状态
- 再查 ECS CPU、内存、磁盘 IO
- 再看 OBS 静态资源访问是否正常

## 恢复操作

- 数据库恢复优先使用 RDS 自动备份
- 应用层恢复优先使用标准镜像或初始化脚本
- 每次变更记录到操作记录与排障文档
